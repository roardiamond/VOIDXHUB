# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Yes            |
| Older   | ❌ No             |

## Reporting a Vulnerability

Agar aapko koi **security vulnerability** dikhti hai toh please turant report karo. Hum ise seriously lete hain.

### Reporting Process

1. **Email bhejo** → `voidxhub.security@gmail.com`
2. Ya **Telegram** pe message karo → `@YASHXCHI` ya `@VOIDXHUB_ADMIN`

**Please include following details:**

- Vulnerability ka type (XSS, SQLi, Payment Bypass, etc.)
- Step-by-step reproduction steps
- PoC (Proof of Concept) agar possible ho
- Impact (Kya damage ho sakta hai)
- Suggested fix (optional)

## Response Time

- **Critical Issues** (Payment bypass, Admin access, etc.) → **Within 24 hours**
- **High** → Within 48 hours
- **Medium/Low** → Within 7 days

## Scope

**In Scope:**
- `roardiamond.github.io/VOIDXHUB`
- All payment related pages (`entryfee.html`, `brentry.html`, etc.)
- Telegram bot integration
- User data handling

**Out of Scope:**
- Social engineering attacks
- Physical attacks
- DDoS attacks
- Spam / Abuse
- Issues in third-party services (Tailwind, Google Fonts, etc.)

## Do Not

- Public mein vulnerability disclose mat karo (responsible disclosure follow karo)
- Production data ko access ya modify mat karo
- Brute force, flooding, ya destructive testing mat karo

## Thank You

Tere jaise researchers ki wajah se hum apne platform ko secure rakh paate hain.  
Valid vulnerability report karne walon ko **reward / shoutout** bhi mil sakta hai.
